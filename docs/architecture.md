# Архитектура проекта

## Компоненты

### 1. Browser Extension

Расширение встраивается в страницу маркетплейса, извлекает идентификатор товара и продавца, отправляет запрос в backend и показывает пользователю:

- надежность продавца;
- достоверность рейтинга;
- признаки риска;
- итоговую рекомендацию.

### 2. Go Backend

Go-сервис выполняет роль API Gateway и orchestration-слоя:

- принимает запросы от расширения;
- читает исторические данные из PostgreSQL;
- использует Redis для кеша;
- вызывает Python-сервис аналитики;
- собирает итоговую оценку надежности.

### 3. Python ML Service

Отдельный сервис анализирует признаки накрутки:

- аномально резкий рост отзывов;
- слишком высокая доля пятизвездочных оценок;
- однотипность текстов;
- сильное расхождение между рейтингом и долей возвратов/жалоб.

### 4. PostgreSQL

Хранит:

- продавцов;
- товары;
- отзывы;
- агрегированные метрики;
- историю score.

### 5. Redis

Кеширует результаты анализа на короткий срок, чтобы повторно не пересчитывать одну и ту же карточку.

## Логика trust score

Итоговая оценка в диапазоне 0..100 строится на основе:

- базовой репутации продавца;
- доли успешных заказов;
- доли споров и возвратов;
- стабильности рейтинга;
- результата антифрод-анализа;
- качества отзывов.

Пример интерпретации:

- `80-100` - высокий уровень доверия;
- `60-79` - умеренный риск;
- `0-59` - высокий риск.
